# Deployment Terms

Deployment of Resonance Docker images is not a transfer of ownership or source code rights.

Unless a separate signed agreement says otherwise, the customer receives limited internal-use rights to run the delivered Docker images for the agreed deployment.

The following actions are prohibited without explicit written permission:

- redistribution or resale;
- modification of images or application artifacts;
- reverse engineering, decompilation, or extraction of image layers for source recovery;
- publishing images, configuration templates, or deployment artifacts to third parties.

Docker images reduce casual source disclosure, but they are not full protection from reverse engineering. Any secret that has ever appeared in local `.env` files or development logs must be rotated before production release.
